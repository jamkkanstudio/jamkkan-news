# Sprint 12 — 핵심 기능 UX 점검 및 1차 디자인 기반 정리

## 상태

구현 완료 · 배포 검증 대기

## 목표와 범위

홈, 뉴스 카드·브리핑, 로그인·관리자 경계, 관심 분야, 성장, 분석의 핵심 흐름을 읽기 전용으로 점검하고 기능·데이터 모델을 바꾸지 않는 1차 디자인 기반을 적용합니다. 익명 공개 읽기, legacy 성장 흐름, `ADMIN_EMAILS` 페이지·서비스 경계, 자동 수집, JSON·Supabase 동기화, 사용자별 격리는 유지합니다. 썸네일 수집, AI 태그·분류, 유료 API, 대규모 프런트엔드 이전과 기존 데이터의 삭제·재작성·소유권 변경은 제외합니다.

## UX 위험과 결정

- 모바일 첫 화면이 긴 성장 요약으로 채워져 뉴스 진입이 늦고, 데스크톱의 기사 10개도 한 줄 목록이라 비교가 어려웠습니다. 홈 성장 요약은 오늘 진행률만 남기고 TOP5는 데스크톱 2열·모바일 1열로 배치합니다.
- 넷플릭스·왓챠처럼 여러 콘텐츠를 한눈에 보는 구조는 참고하되, `BRAND.md`와 `PRINCIPLES.md`의 차분함·여백·시간 투자 원칙에 맞춰 밝고 절제된 팔레트와 제한된 카드 수를 유지합니다.
- 주제 표시는 저장된 카테고리만 사용합니다. 현재 자동 수집 기사에 `기타`가 많은 품질 문제를 숨기기 위해 임의 분류를 만들지 않으며, AI 태그와 썸네일 공급은 별도 데이터 결정 전까지 추가하지 않습니다.
- Streamlit 기본 컴포넌트로 반응형 그리드와 공통 디자인은 가능하지만 정교한 가로 캐러셀·전환·제스처는 한계가 있습니다. 이번 Sprint에서는 프레임워크를 바꾸지 않고 실제 모바일 결과를 먼저 확인합니다.
- 기본 멀티페이지 메뉴가 사용자 화면에 관리자 항목과 `app` 이름을 함께 보여줬습니다. 사용자 메뉴를 별도로 제공하고 관리자 링크는 `is_admin()`일 때만 표시하되, 페이지와 서비스의 기존 권한 검사는 그대로 둡니다.

## 구현 결과

- 공통 색상·타이포·간격·카드·버튼·상태 메시지와 Streamlit light theme을 추가했습니다.
- 홈 TOP5와 나의 TOP5를 반응형 카드 그리드로 바꾸고 제목·카테고리·3줄 요약·30초 브리핑 행동의 위계를 통일했습니다.
- 홈 성장 요약을 한 카드로 줄이고, 성장·분석의 오늘/누적 지표와 7일 기록을 모바일에서도 한눈에 보이는 구조로 정리했습니다.
- 홈·설정·성장·분석·관리자 화면의 제목과 설명을 같은 규칙으로 통일했습니다.
- 사용자 내비게이션과 계정 상태 문구를 정리하고, 비관리자에게 뉴스 관리·수집 링크를 숨겼습니다. 관리자 페이지 차단과 로그인 필요 메시지도 더 구체적으로 바꿨습니다.

## 검증

- 표적 테스트: `python -m unittest -v test_design_system test_auth_service` — 15개 통과
- 최종 코드 변경 뒤 `python -m unittest discover -v` — 71개 통과
- Python 전체 컴파일과 `.streamlit/config.toml` 파싱 통과
- 로컬 Streamlit `/_stcore/health`와 `/` — HTTP 200
- 브라우저 390px 점검에서 홈 1열, 짧은 성장 요약, 성장·분석 7일 스트립, 설정·뉴스 수집 차단 메시지를 확인했습니다.
- 기본 폭 점검에서 TOP5 2열과 첫 카드의 브리핑·원문·30초 투자 행동을 확인했고 브라우저 콘솔 오류는 0건이었습니다.
- `git diff --check` 통과, 변경 파일의 credential 패턴 0건, tracked `.env`·`.streamlit/secrets.toml` 0건을 확인했습니다.
- `news.json`은 커밋 전 동기화한 원격 자동 수집 커밋 `981073f`, `8d202c8`, `8c9631b`의 내용과 일치합니다. `events.json`, `growth.json`, `interest.json`, `settings.json`은 Sprint 시작 해시와 같고 다섯 보호 파일 모두 작업 트리 diff가 없습니다.
