# Sprint 02 — Supabase 신규 뉴스 병행 저장

## 상태

완료 · 2026-07-15

## 목표

기존 JSON 저장을 유지하면서 신규 뉴스 한 건을 동일한 UUID로 Supabase에도 저장합니다.

## 범위

포함:

- 수동 뉴스 추가
- RSS 선택 뉴스 추가
- JSON 저장 후 Supabase `news` 테이블 upsert
- 병행 저장 자동 테스트와 실제 배포 검증

제외:

- 뉴스 수정·삭제 Supabase 동기화
- 기존 JSON 뉴스 전체 이전
- 성장·분석·설정 데이터 이전
- 로그인과 사용자별 데이터 분리

## 구현 결과

- 신규 뉴스에 생성한 UUID를 JSON과 Supabase에서 동일하게 사용
- Supabase 저장 성공·실패를 반환하도록 저장 서비스 보완
- Supabase 실패 시 JSON 저장을 유지하고 서버 로그와 화면 경고 제공
- 수동 추가와 RSS 일괄 추가에서 저장 결과를 사용자에게 표시
- 실제 배포 환경에서 JSON·Supabase 동시 저장 확인
- 검증용 데이터 정리 완료

## 검증

- 자동 단위 테스트 5개 통과
- Streamlit 배포 앱 정상 기동 확인
- Supabase 연결 및 `news` 테이블 조회 확인
- 테스트 뉴스의 JSON·Supabase 동일 UUID 확인
- 운영 secret 키 교체 후 기존 키 폐기와 재연결 확인
