# Sprint 05 — KST 날짜 기준 통일

## 상태

구현 완료 · 배포 검증 대기 · 2026-07-15

## 목표

성장 기록, 연속 성장, 주간 화면과 분석 화면의 사용자 날짜 판단을 명시적인 `Asia/Seoul` 기준으로 통일합니다.

## 범위

포함:

- Python 표준 라이브러리 `zoneinfo` 기반 KST 날짜·시간 유틸리티
- 성장 기록 저장·조회, 오늘 브리핑 완료와 연속 성장의 날짜 통일
- 성장 및 분석 주간 범위의 날짜 통일
- 분석 이벤트 시각의 KST 날짜 변환
- UTC/KST 자정 경계 회귀 테스트

제외:

- Google 로그인과 사용자별 데이터 분리
- Flutter 전환
- 기존 JSON 및 Supabase 데이터 수정·삭제·재백필
- 사용자 날짜 판단과 무관한 뉴스 생성·RSS 수집 타임스탬프 변경

## 구현 결과

- `services/time_service.py`에 `Asia/Seoul` 기준 현재 시각, 오늘 날짜, 이번 주 날짜와 시각→KST 날짜 변환을 모았습니다.
- 성장 서비스의 오늘 기록 저장·조회, 중복 읽기, 브리핑 완료와 연속 성장 계산이 같은 KST 날짜를 사용합니다.
- 성장 페이지의 오늘 카드와 주간 날짜가 공통 KST 함수를 사용합니다.
- 분석 이벤트 저장 시각과 이번 주 기간 계산도 같은 공통 함수를 사용합니다.
- timezone 정보가 없는 과거 이벤트는 기존 동작과 호환되도록 KST 시각으로 해석합니다.
- `news_service.py`와 `rss_service.py`의 타임스탬프는 사용자 날짜별 판단에 쓰이지 않아 범위 밖으로 유지했습니다.

## 데이터 보존

`data/growth.json`을 포함한 기존 JSON 파일과 Supabase 데이터는 변경, 삭제 또는 재백필하지 않았습니다. 2026-07-14와 2026-07-15 기록은 원본 그대로 보존됩니다.

## 검증

- 자동 단위 테스트 24개 통과
- UTC 2026-07-14 15:30이 KST 2026-07-15로 판단되는 경계 확인
- KST 다음 날 기록 시 연속 성장이 이어지는지 확인
- UTC 타임스탬프 이벤트가 KST 기준 이번 주에 포함되는지 확인
- 기존 naive 이벤트의 KST 호환 해석 확인
- Streamlit 배포 검증: 대기
