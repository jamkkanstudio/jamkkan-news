# Sprint 15 — 무료 핵심 요약 품질 개선

## 상태

완료

## 목표와 범위

유료·생성형 API 없이 신규 수집 기사 카드의 브리핑 품질을 높입니다. 중복 판정을 통과한 기사만 수집 시 공개 원문을 한 번 읽고, 출처 노이즈를 제거한 뒤 원문 문장 1~2개를 합계 140자 이내로 선택합니다. 자동 수집과 관리자 수동 복구가 같은 요약 경로를 사용하며, 원문·파싱·품질 실패는 기사별 RSS 설명으로 격리합니다.

사용자 요청 시점 요약, 생성형 `왜 중요한가`, 기존 기사 backfill, 기존 JSON·Supabase 행 재작성, 원문 전문 저장, 관심사·체류시간 학습, 유료 API와 새 NLP 의존성은 제외했습니다. 익명 공개 읽기, legacy 성장, KST 고정 후보와 첫 5개 완료 의미, 다음 최대 5개, 도움 신호, `ADMIN_EMAILS` 페이지·서비스 경계, 사용자 격리를 보존했습니다.

## 감사·출처 경계·데이터 결정

- 자동 경로는 `news_collection_service._build_news()`, 관리자 복구는 페이지 내부에서 각각 RSS 설명을 요약하고 있었습니다. 둘을 공통 서비스로 합쳤고, 중복 판정 뒤 실제 신규 기사에 대해서만 원문을 요청합니다.
- 운영 RSS URL과 기존 신규 행은 `https://www.yonhapnewstv.co.kr/news/{기사ID}` 형태입니다. 2026-07-15 확인한 `robots.txt`는 `/news/`를 차단하지 않고 검색·제보·게시판 경로를 제외합니다. 공식 RSS 안내는 영리 목적 배포·사용을 금지하고 사이트는 무단 전재·재배포·AI 학습을 금지하므로, 현재 비영리 RSS 운영 전제에서 공개 기사 한 건을 일회성으로 읽고 140자 이하 원문 문장과 원문 링크만 제공하는 최소 범위로 제한했습니다.
- HTTPS, 정확한 호스트와 `/news/{기사ID}` 경로, 최종 리디렉션을 재검증하고 6초 timeout, 512KB 상한, 식별 가능한 User-Agent를 적용했습니다. 다른 출처·경로는 네트워크를 열지 않고 RSS 설명을 사용합니다.
- 원문 전문이나 중간 정제 결과는 JSON·Supabase·상태·로그에 저장하지 않습니다. 기존 `summary` 필드만 바뀌므로 스키마·기존 행·소유권 마이그레이션이 없습니다. 코드 롤백은 이전 RSS-only 요약으로 돌아가며 데이터 롤백이나 기존 행 삭제가 필요 없습니다.

## 구현 결과

- 표준 라이브러리 HTML 파서로 연합뉴스TV `article-body-text`만 추출하고 앵커·기자 표기, 기자명·이메일, 영상·편집·그래픽 크레딧, 사진 설명, 저작권·재배포 문구, 제목 반복과 중복을 제거합니다.
- 제목 단어, 기사 앞부분, 수치와 보도 동사를 결정적으로 점수화해 원문 문장 1~2개를 합계 140자 이내로 선택합니다. 문장을 생성하거나 원문에 없는 사실을 보태지 않습니다.
- 원문이 짧거나 문장이 부족한 경우와 timeout·크기·형식·파싱 오류는 기존 RSS 설명으로 fallback합니다. 방어 경계를 두 겹으로 둬 한 기사 예외가 다음 기사나 전체 수집을 막지 않습니다.
- 자동 수집은 중복 기사 원문을 요청하지 않으며, 관리자 복구는 기사 선택·등록 시 같은 공통 함수를 사용합니다. 카드 클릭과 익명 공개 읽기는 저장된 `summary`만 읽습니다.

## 배포 상태

자동 수집 데이터 커밋 `5dd760d`, `a084da5`를 보존한 코드·문서 커밋 `8b6faa5`를 `main`에 push했습니다. 운영 익명 홈에서 기존 첫 5개, `NEW`, 다음 최대 5개와 원문 읽기 흐름이 유지되는 것을 확인했습니다. 관리 직접 URL은 익명 사용자에게 관리 UI를 노출하지 않고 홈으로 돌아갔으며 브라우저 오류·경고는 0건이었습니다.

기존 기사는 backfill하지 않는 범위이므로 운영 카드의 과거 RSS 요약은 변경하지 않았습니다. 새 원문 경로는 push 전 같은 공개 기사에 실제 네트워크 요청을 보내 fallback 없이 140자 이내 원문 두 문장을 생성하는 것으로 검증했고, 운영 브라우저에서는 데이터 쓰기 CTA를 실행하지 않았습니다. 다음 자동 수집부터 신규 행의 기존 `summary` 필드에만 적용됩니다.

## 다음 경계

출처 이용 조건이 바뀌거나 영리 운영을 검토하면 원문 추출을 먼저 중지하고 별도 사용 허가와 정책을 확인합니다. 다른 언론사 확대도 각 출처의 robots·이용 조건·HTML 구조·fixture·fallback 검증 없이 allowlist에 추가하지 않습니다. 기존 기사 backfill과 생성형 요약은 별도 데이터·비용·저작권 결정 전에는 수행하지 않습니다.
