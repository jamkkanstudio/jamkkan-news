## 운영 원칙

1. Sprint마다 하나의 명확한 목표를 정합니다.
2. 한 단계 구현 후 테스트하고 다음 단계로 넘어갑니다.
3. 완료 시 관련 테스트, 커밋, 배포 결과를 기록합니다.
4. 제품 우선순위는 `ROADMAP.md`, 실제 변경은 `CHANGELOG.md`를 기준으로 확인합니다.
